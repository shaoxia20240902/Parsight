## 测试前置条件

- 至少存在一个状态为 `analyzed` 或 `understanding_ready` 的 Excel 文件。
- 数据理解模块已完成，Sheet 元数据、关键维度、关键指标可读取。
- BI 模块已有配置时，生成 BI 链路可测试“匹配已有图表”和“新建图表”；无已有配置时，应走新建方案。

## 用例 1：快速问答

测试题：
“销售额最高的前 5 个客户是谁？”

预期结果：
- 首次发送后不直接执行查询，而是出现“意图识别阶段”确认卡。
- 确认卡显示模式为“快速问答”，包含置信度、命中依据、执行步骤和三个动作：继续执行、我补充一下、先不执行。
- 点击“继续执行”后，系统调用快速问答链路。
- 回复中展示“智能体执行轨迹”，至少包含意图识别、字段定位、数据查询、答案生成。
- 如果 SQL 成功，回复中展示查询结果表和推荐追问。

## 用例 2：深度洞察

测试题：
“为什么华东区域最近的销售额增长放缓？请从客户、产品和时间趋势几个角度分析。”

预期结果：
- 首次发送后出现“深度洞察”的意图确认卡。
- 点击“继续执行”后进入 SSE 深度分析进度。
- 回复持续更新进度，包含关键词澄清、多角色问题拆解、子问题筛选、查询生成、数据验证、证据图表生成、报告生成。
- 最终输出一段结构化报告，能够说明关键发现、可能原因和后续建议。
- 聊天历史中保留本轮用户问题和最终报告。

## 用例 3：生成 BI

测试题：
“帮我生成一个客户销售表现看板，要有客户销售额 Top 10、尾部客户和月度趋势。”

预期结果：
- 首次发送后出现“生成 BI”的意图确认卡。
- 点击“继续执行”后进入 BI Builder 链路。
- 如果已有相似图表，优先提示已有候选，并允许调整已有或新建。
- 如果没有相似图表，生成报表方案，展示将创建的图表列表。
- 如识别到业务知识或偏好，弹出确认业务知识窗口。
- 点击确认生成后，BI 图表写入看板，并返回“去查看”类动作。

## 用例 4：补充意图

测试题：
“看一下这个。”

预期结果：
- 确认卡显示低置信度或歧义提示，例如“问题较短”或“包含指代词”。
- 点击“我补充一下”后，原问题回填到输入框。
- 用户补充指标、维度或时间范围后，可重新触发意图识别。

## 用例 5：取消执行

测试题：
任意问题，例如“各区域利润率怎么样？”

预期结果：
- 出现意图确认卡后，点击“先不执行”。
- 系统回复已停止执行。
- 不调用快速问答、深度洞察或 BI Builder 的执行接口。
