超级问答智能体：快速问答、深度洞察、生成 BI

三种 Chat 类型共用“意图识别与用户确认”入口。用户提出问题后，系统先判断是否继续、适合哪条执行链路、命中哪些数据依据以及可能的歧义；用户确认后才进入真实查询、深度分析或 BI 写入流程。

前端确认卡 ReAct 深度链路 BI 写入链路 用户可中断/补充

统一交互主流程

1

用户输入问题

用户在快速问答、深度洞察或生成 BI 中输入自然语言需求。

2

装配数据上下文

读取文件状态、Sheet 摘要、字段类型、关键维度和指标。

3

意图识别

判断模式、置信度、命中字段、潜在歧义和推荐执行计划。

4

用户确认

前端展示确认卡：继续执行、补充一下、先不执行。

5

进入模式链路

根据用户确认进入快速问答、深度洞察或生成 BI。

6

输出结果

回答、报告、数据表、图表方案或 BI 看板写入结果。

7

沉淀上下文

保存聊天历史、业务知识、用户偏好和后续追问入口。

三种问答链路

快速问答

  1. 识别问题指向。
  2. 选择相关 Sheet 和字段。
  3. 必要时生成安全 SELECT 查询。
  4. 返回简洁结论、查询结果表和推荐追问。

深度洞察

  1. 关键词和筛选条件识别。
  2. 业务分析师、数据分析师、管理决策者多角色拆解。
  3. 筛选子问题并生成 SQL。
  4. 执行查询、生成图表证据。
  5. 输出结构化深度报告。

生成 BI

  1. 识别创建、修改、探索或非 BI 意图。
  2. 匹配已有图表，避免重复创建。
  3. 规划指标、维度、图表类型和分类。
  4. 确认业务知识、偏好和报表方案。
  5. 写入 BI 看板并返回查看入口。

接口与前端契约

POST /api/chat/intent 返回 mode、intent、confidence、evidence、warnings、plan_steps 和 actions，用于前端意图确认卡。
POST /api/chat/quick-qa 确认后执行快速问答，返回 answer、data、recommended_questions 和 agent_steps。
POST /api/chat/deep-research SSE 返回深度链路进度，前端实时展示智能体执行轨迹和最终报告。
POST /api/chat/bi-builder 确认后进入生成 BI 链路，继续使用已有问卷、知识确认、方案调整和生成动作。
