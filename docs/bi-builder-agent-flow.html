BI 构建者智能体流程图

这张图描述问答页中 BI 构建者从用户提问、意图判断、已有候选、ScopePlanning、业务知识与偏好确认、统一确认到最终生成并跳转 BI 页的完整流程。

意图与上下文 复用已有看板 用户交互补全 生成与写入 阻塞或失败

1用户提问

用户在聊天页选择“构建者”模式，并输入自然语言需求。

  • 支持推荐问题点击
  • 支持补充输入再次进入流程

2意图判断

判断是否属于 BI 查看、创建、调整、补充或数据探索。

  • 非 BI：返回默认能力说明
  • 识别 force_create 和 explore_dataset

3加载上下文

读取表理解、字段画像、BI 分类、现有图表、筛选器。

  • 检查文件和六维理解状态
  • 整理可选字段和分类

4可用性门禁

如果数据或理解未就绪，不能继续创建。

  • 提示原因
  • 给出去数据页查看进度按钮

5是否跳过复用

用户明确新建时直接进入 ScopePlanning，否则才查已有图表候选。

  • 必须新建：跳过匹配
  • 数据探索：主动推荐方向

6匹配已有候选

按指标、维度、意图、筛选、分类语义评分，只产出候选。

  • 不替用户直接决定
  • 最多返回 3 张候选图

7候选动作

找到已有图表时给用户三个清晰动作。

  • 去查看
  • 基于此图调整 / 我要新建

8ScopePlanning

一次性完成需求探索、搭配推荐、批量范围和缺失信息判断。

  • 输出 readiness
  • 输出 chart_list

9知识/偏好候选

需求探索清楚后，再识别业务知识和用户偏好。

  • 神马 = 神州数码
  • 排名默认用条形图

10确认卡片

用户确认后才写入业务词典、表理解或用户偏好。

  • 确认添加 / 本次使用
  • 不保存 / 我来改

11必填问卷

只展示继续生成必需的问题，减少用户负担。

  • 通常 1-2 个问题
  • 确认并继续

12高级设置

可选项折叠展示，不阻塞主流程。

  • 排序、Top N、颜色等
  • 默认可跳过

13图表列表

单图和多图统一用 chart_list，搭配建议可增删。

  • 单图 = 列表长度 1
  • Top 5 可加入 Bottom 5

14统一确认摘要

字段、筛选、图表类型、推荐分类和写入策略一次确认。

  • 多图用 Markdown 表格
  • 表格图也用 Markdown

15分类调整

分类依赖最终字段和跨表关系，在摘要中一起调整。

  • 不同意可重新选择
  • 分类少于 8 个可新建

16影响范围确认

整分类重做或替换已有图表时，必须二次确认影响范围。

  • 显示将替换哪些图
  • 保留回滚快照

17最终确认

用户点击“确认生成”后才进入真实创建。

  • 可返回修改 chart_list
  • 可返回修改字段/分类

18生成配置

复用 BI SQL builder、图表配置模型和筛选器规则。

  • 生成 SQL
  • 生成图表 config

19预览校验

执行 SQL 预览，确保图表数据不是编造结果。

  • 成功：写入配置
  • 失败：返回修正选项

20写入并展示

新图表或批量图表进入目标分类和图表仓库，聊天中返回预览。

  • 按钮：去查看
  • 定位到 BI 页面图表

候选不是决定

已有图表只能作为候选返回，用户可以去查看、基于此图调整，或选择完全新建。

交互优先选项化

指标、维度、筛选、分类、图表类型都优先从当前数据上下文里生成选项，保留“其他”作为兜底。

创建前必须确认

最终生成前要一起确认字段、指标口径、筛选条件、推荐分类和写入策略。表格类必须用 Markdown 表格呈现。

ScopePlanning 一次完成

需求探索、搭配推荐、批量范围、缺失信息和推荐分类在一次规划里完成，减少反复调用和前后推翻。

知识和偏好后置

业务知识和用户偏好在需求探索清楚后确认，避免在明确新建且无模糊词时浪费调用。

支持批量和重做

支持一次创建多张图，也支持重做整个分类下的 BI；影响已有图表时必须展示范围并二次确认。

前后端职责链路

前端聊天页
发送消息→ 渲染富消息→ 按钮/问卷提交→ 图表预览
构建者接口
状态机→ 意图判断→ 已有图表匹配→ ScopePlanning→ 知识/偏好确认→ 统一确认
BI 服务层
字段画像→ SQL 编译→ 预览校验→ 写入 BI 配置
