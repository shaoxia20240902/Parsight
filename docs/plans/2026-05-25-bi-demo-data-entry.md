# BI 演示数据与入口选择优化

## 背景

BI 页面在用户尚未上传文件时不应该是空白页。对首次进入的用户，页面需要直接展示一套可交互的演示看板，并明确提示当前为演示数据。对已经上传文件且表理解全部完成的用户，每次进入 BI 页面时需要先给出选择：查看演示数据，或启动 AI 全局构建。

演示数据参考 `mock_data/企业销售数据.xlsx`，包含 5 个默认 Sheet 和 3 个自定义主题分类。每个分类展示 8 张图表，其中单独 KPI 卡片按一张图表计算。

## 交互规则

1. 无上传文件：直接展示演示看板，不弹窗；页面顶部显示“当前展示演示数据”的轻提示，并提供前往数据页入口。
2. 已上传文件但表理解未完成：不展示演示弹窗，提示用户等待数据理解完成。
3. 已上传文件且表理解完成：每次进入 BI 页面弹出选择框。
4. 选择“查看演示数据”：切换到演示看板，所有操作仅在前端本地生效，不请求后端。
5. 选择“AI 全局构建”：进入真实文件的 BI 生成流程；若已有 BI 配置，则按重新生成处理。

## 演示看板结构

分类总数固定为 8 个：

- 默认 Sheet：销售明细、销售员信息、产品信息、区域目标、客户信息
- 自定义分类：经营总览、目标达成、客户产品

每个分类 8 张图表，共 64 张。图表类型覆盖 KPI、柱状图、横向柱状图、折线图、面积图、饼图、环图、组合图、排名图、明细表、矩形树图、漏斗图、散点图、气泡图、热力图、雷达图、仪表盘、瀑布图等 20+ 类型，保证用户进入演示页时能看到完整能力边界。

## 技术实现

- 新增 `frontend/src/mocks/demoBIConfig.ts`，输出与后端 BI 配置兼容的演示配置。
- `BIView.vue` 负责区分无文件、理解中、真实文件三种入口状态。无文件时直接渲染演示看板。
- `BIDashboard.vue` 负责真实文件已就绪后的选择弹窗，以及演示/真实生成的切换。
- `BIInsightsBoard.vue` 增加 `demoMode`，演示模式下筛选、编辑、仓库移动只改本地状态，不触发图表数据刷新或保存 API。
- `BIWarehouseModal.vue` 增加 `demoMode`，演示模式下编辑图表只回写前端状态。

## 验收点

- 没有上传文件时，访问 `/bi` 直接看到 8 个分类、64 张图表的演示看板。
- 演示看板顶部有清晰但不打断流程的演示数据提示。
- 上传文件且表理解完成后，每次进入 `/bi` 都出现两个按钮的选择弹窗。
- 点击“查看演示数据”不会触发任何 BI 后端生成或图表查询接口。
- 点击“AI 全局构建”进入现有分阶段生成体验。
- 演示模式下编辑图表标题、移动仓库、筛选等交互不报错。
